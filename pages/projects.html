Interactive Fiction Adventure Game

Description

This project is a Command-Line Interface (CLI) fictional game created in Python. I modeled this from the world renowned game Dungeons & Dragons (DND). You become so enthrolled with the interactive game-play that you forget you're only using the terminal!

The game first takes you through the steps of building your adventurer based on race and class. Then, it gives you a breakdown of skills your character could have. Which leads you to the next phase of increasing your skills by rolling 3 6-faced dice, totaling the 2 highest values, and adding the total to your skills' point-bases already calculated by the game depending on your race and class. After displaying your complete skillset, you can begin your adventure!

The game walks you through an interactive story-line where you have options to choose from that change the outcome of your adventure. If you don't choose wisely, you may never find your fellow adventurers or, even worse, lose your life!

Contact Management System

Description

This project is a Command-Line-Interface (CLI) application built to simplify the management of your contacts. It was created to be user-friendly and has an extesive menu of options.

You can choose from:

  1. Add a New Contact
  2. Edit an Existing Contact
  3. Delete a Contact
  4. Search for a Contact
    • Generates search options based on the first contact's keys
  5. Display All Contacts
    • Generates sorting options based on the first contact's keys
    • If no sort option was selected, the default sorted option is by name
  6. Export Contacts to a Text File
    • Exports the contacts in a structured, user-friendly format
    • Ensures there are no duplicated contacts
  7. Import Contacts from a Text File
    • Imports contacts from a text file
    • Uses REGEX patterns to match for proper importing
    • Utilizes a merge function to ensure there are no duplicated contacts or values
  8. Restore Contacts from Backup File
    • Calls to a find_latest_backup function to find the most recent backup file to read from
    • Updates the contacts dictionary with the latest backup details
  9. Quit

It has implementations for automatic importing of contacts from a text file (if it exists) and creating a NEW backup file within a Backup Directory. There are also many validation and formatting features that match user inputs to patterns.

Basic Library Management System

Description

This project is a Command-Line_interface (CLI) based application to streamline the management of users, books, and resources within a library. This program was built using Python Programming Language while applying Object-Oriented Programming (OOP) principles and implementing an enhanced User Interface. It is a robust system that allows users to browse, borrow, return, and explore a collection of books.

There are 4 different menus with many options to choose from!

Main Menu:

  1. User Operations
    • Sends the user to the User Operations Menu
  2. Book Operations
    • Sends the user to the Book Operations Menu
  3. Author Operations
    • Sends the user to the Author Operations Menu
  4. Quit

User Operations Menu:

  1. Add a New User
    • Generates a unique Library ID, ensuring no duplicated IDs
    • Asks user input for name
    • Adds the new user to the library
  2. View User Details
    • Prompts the user for Library ID to view details
    • Validates/Formats/Revalidates(if necessary) the Library ID
    • Provides the user's details in a user-friendly format
  3. Display All users
    • Provides the user a title for the list
    • Prints each user's Library ID & Name
  4. Return to Main Menu
    • Sends the user back to the main menu

Book Operations Menu:

  1. Add a New Book
    • Prompts the user to enter title, author, genre, and publication date for the new book
    • Validates the publication date ensuring the year already exists
    • Adds the new book to the library
    • If the author is a NEW author, sends the user to an add_author function
  2. Borrow a Book
    • Prompts the user for title of the book to borrow and checks if it exists
    • If the book exists, the user is prompted for Library ID which is then validated and formatted/revalidated (if needed)
    • Then it adds the book to the user's borrowed books list and marks it as 'Borrowed' in the library
  3. Return a Book
    • Prompts the user for title of the book to return and their Library ID, which is then validated and formatted/revalidated (if needed)
    • Then it removes the returned book from the user's borrowed books list and marks it as 'Available' in the library
  4. Search for a Book
    • Prompts the user for title of the book to search for and advises the user of the action
    • Then it finds the book details within the Library and returns the results
  5. Display All Books
    • Returns all the books' details in a user-friendly format
  6. Return to Main Menu
    • Sends the user back to the main menu

Author Operations:

  1. Add a New Author
    • Prompts the user for the author's name
    • After the name is collected, the user is prompted for the author's biography
    • Then it adds the new author to the library
  2. View Author Details
    • Prompts the user for the author's name and advises of the action
    • Returns the author's details in a user-friendly format
  3. Display All Authors
    • Provides the user a title for the Authors List
    • Returns a list of each author's details in a user-friendly format
  4. Return to Main Menu
    • Sends the user back to the main menu

It has implementations many validations and formatting features that match user inputs to patterns.

e-Commerce API

Description

This project is a Flask-based E-Commerce API that provides a comprehensive set of features for managing customers, products, and orders. The application integrates with a MySQL database using Flask-SQLAlchemy and Marshmallow for data modeling and validation.

It has many features including:

  1. Customer & CustomerAccount Management
    • Implemented CRUD (Create, Read, Update, Delete) endpoints for managing customers and their associated accounts
    • Capture essential customer information, including name, email, and phone number
    • Stores and manages customer account details, including username and password
    • Additional security measures in place for salting and hashing customer's passwords
  2. Product Catalog Management
    • Implemented CRUD endpoints for managing products in the e-commerce catalog
    • Captures product details such as name and price
    • Provides functionality to list all available products
    • View and manage product stock levels
    • Automatically restocks products when stock levels fall below a specified threshold
  3. Order Processing
    • Implemented endpoints for customers to place new orders, specifying the products (either by product ID and/or name) and quantities
    • Captures essential order details, including order date and associated customer
    • Allows customers to retrieve order details and track the status of their orders
    • Provides customers access to their order history
    • Calculates the total price of each order based on the associated product prices
  4. Database Integration
    • Utilizes Flask-SQLAlchemy to integrate a MySQL database into the application
    • Creates the necessary database models to represent customers, orders, products, and customer accounts
    • Established proper relationships between the database tables
    • Drawio file with Entity Relation Diagram (ERD) for database tables
  5. Data Validation & Error Handling
    • Implemented data validation mechanisms to ensure user inputs meet specified criteria
    • Implemented try & except blocks to handle errors gracefully and provide informative error messages
  6. User Interface (Postman)
    • Postman collections to organize and group API requests by functionality
    • Separate collections for Customer Management, Product Management, Order Management, and Additional Features

Additional file for testing examples. All dependencies were installed within a virtual environment. As well as, any prerequisites can be utilized through extensions in Visual Studio Code.

Card Fraud Transaction Simulator

Description

This project produces legitimate and fraudulent card transactions. It can be used to assess the efficiency of different fraud detection techniques in a reproducible way. It is especially helpful when needing to complete data analysis and risk assessment.

There are many features including:

  1. Base Customer Profiles Table
    • Generates customer properties from random distributions
  2. Base Terminal Profiles Table
    • Generates terminal properties from random distributions
  3. Base Available Terminals
    • Creates a list of terminals within a radius (50)
    • Creates a figure of the available terminals to a customer using a scatter plot and a circle indicating the radius
    • Adds a new column in the customer profile table for available terminals
  4. Base Transactions Table
    • Generates a transactions table for customers from a set start date to a number of days
    • This includes the date/time, terminal ID, transaction amount, and more
  5. Larger Dataset for Terminals & Transactions
    • 5000 Customers
    • 10,000 Terminals
    • 90 Days of Transactions
    • Corresponding dates from 2023/12/01 - 2024/02/28
    • Generates 2,156,243 Transactions
  6. Plotting Distribution of Transaction Amounts & Times
    • Histograms for transaction amounts and times
    • Labeled and color coded for a user-friendly format
  7. Fraud Scenarios
    • Scenario 1
      • Registers possible fraud if the transaction amount is larger than X
      • Increases the probability of fraud by a certain decimal point depending on the amount
      • Adds the probability to that specific customer's transaction row for further analysis
    • Scenario 2
      • Registers possible fraud if criminal use of random terminals via phising methods
      • For instance: Compromised gas station terminal, online terminal, or other methods
      • Increases the probability of fraud by 1 if the terminal is compromised
      • Adds the probability to that specific customer's transaction row for further analysis
    • Scenario 3
      • Registers possible fraud if a random customer has unusual transaction amounts
      • Shows 1/3 of transactions' amounts multiplied by 5 over 14 days
      • Increases the probability of fraud by 2 when this occurs
      • Adds the probability to that specific customer's transaction row for further analysis
  8. Visualizations of Data
    • Line graph showing total transactions, number of fraudulent transactions, and number of compromised cards per day
    • Bar graph showing total transactions, number of fraudulent transactions, and number of compromised cards per day
    • Additional graphs for better clarity of details where the total transactions are divided by 50
  9. Data Sampling
    • Takes an initial fraction of data as a sample and creates a new dataframe
    • From the new dataframe, take another fractional smapling to use for export to CSV and Excel files for analysis & assessment
    • Each sampling can be repeated to show a different sampling for analysis & assessment
  10. Risk Assessment
    • The data exported to CSV & Excel can then be used in the RiskAssessment Excel file to show the risk evaluation
    • Using a risk matrix, the cells will be color coded based on the evaluation
    • Using the values from the sample data, a graph will also populate that highlights the spike in fraud throughout the transactions
